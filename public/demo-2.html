Try me out with Parcel or WebPack

When using Parcel, we can easily point to this html file as the entry-point

  • Parcel will build everything and put it in a `dist` folder
  • We will need to change our script types from "module"
  • We will need to "defer" or move where our script tags live on the page

When using WebPack, you'll have to configure a lot more yourself. We can't point to this html file (without teaching WebPack how to handle htmls) and so instead we'll point to an entry javascript file, which loads all dependencies for a page.

  • The entry file should reference all dependencies.
  • WebPack will bundle them for you and output a "built" file.
  • We will need to update our html to point to the built file, or teach WebPack how to do that.
